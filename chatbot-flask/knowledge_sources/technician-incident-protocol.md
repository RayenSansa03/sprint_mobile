# Technician Incident Response Protocol

## Intake and triage
- Capture issue category, urgency, and location.
- Verify user profile and historical interventions.
- Classify severity: low, medium, high, critical.

## Response SLA example
- Critical: first response under 2 hours.
- High: same-day intervention.
- Medium: within 24 hours.
- Low: scheduled within 48 hours.

## Closure checklist
- Confirm root cause and corrective action.
- Attach proof (photo, note, or sensor data).
- Record lessons learned for recurring incidents.

## Escalation
- Escalate to admin when SLA breach risk is detected.
